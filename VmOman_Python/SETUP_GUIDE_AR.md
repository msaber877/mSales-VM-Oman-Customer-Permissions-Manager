## الجزء 3️⃣: إدارة ملف الأكواد ديناميكياً (`permission_codes.json`)

ابتداءً من الإصدار 3.1.0، يتم فصل قواعد بيانات الأكواد ووصفها عن الكود المصدري للتطبيق.
- ينشأ ملف `permission_codes.json` تلقائياً في نفس مسار تشغيل التطبيق عند التشغيل الأول في حال عدم وجوده.
- يمكنك تعديل هذا الملف يدوياً باستخدام أي محرر نصوص (مثل Notepad++) بتنسيق JSON قياسي:
```json
{
    "SR_PERMISSION_CODE.a": "Eload",
    "SR_PERMISSION_CODE.b": "DST Channel",
    "Custom_Code_Here": "Custom Description Here"
}
```
- كما يمكنك إضافة وتعديل وحذف الأكواد بالكامل وبطريقة آمنة من خلال واجهة التحرير في التطبيق مباشرة عبر زر **"➕ Add Code"** و **"🗑️ Delete Code"**.

## الجزء 4️⃣: بناء وتوزيع ملف تنفيذي مستقل (`.exe`)

لتوزيع البرنامج على مستخدمين آخرين لا يمتلكون خبرة برمجية أو ليس لديهم بيئة بايثون مثبتة، يمكنك بناء ملف تنفيذي مستقل:

1. تأكد من تثبيت PyInstaller في بيئتك:
   ```powershell
   & "C:\Users\mohamed_saber\AppData\Local\Programs\Python\Python312\python.exe" -m pip install pyinstaller
   ```
2. ابنِ التطبيق باستخدام ملف الإعدادات المرفق `.spec`:
   ```powershell
   & "C:\Users\mohamed_saber\AppData\Local\Programs\Python\Python312\python.exe" -m PyInstaller mSales_Data_Merger.spec
   ```
3. بمجرد انتهاء البناء، ستجد المجلدات التالية قد ظهرت:
   - `build/`: ملفات البناء المؤقتة (يمكن حذفها لاحقاً).
   - `dist/`: يحتوي على الملف التنفيذي النهائي `mSales_Data_Merger.exe`.
4. لتشغيل ملف الـ EXE، قم بنسخه إلى مجلد العمل وقم بتشغيله. سيتعامل تلقائياً مع ملف `permission_codes.json` المتواجد في نفس مجلده.

## الجزء 5️⃣: استكشاف وحل المشكلات الشائعة

### ❌ خطأ: "The file is not a valid ZIP file" عند قراءة ملف Excel
**السبب:** قد يكون ملف الـ Excel تالفاً أو تم حفظه بصيغة قديمة غير مدعومة من مكتبة `openpyxl`.  
**الحل:** افتح ملف الـ Excel الأصلي باستخدام برنامج Microsoft Excel وقم بحفظه مجدداً بصيغة `Excel Workbook (*.xlsx)`.

### ❌ خطأ: "Missing columns: CODE, CF_PERMISSION"
**السبب:** لا يحتوي ملف العملاء المرفوع (CSV) على هذه الأعمدة تماماً بالصيغة المذكورة.  
**الحل:** تأكد من أن أسماء الأعمدة في السطر الأول لملف الـ CSV مطابقة تماماً (بأحرف كبيرة) للاسمين المذكورين.

### ❌ الأحرف العربية تظهر برموز غريبة في ملف التصدير
**السبب:** عدم فتح ملف الـ CSV المصدر بترميز UTF-8.  
**الحل:** البرنامج يدعم التصدير بترميز `utf-8-sig` (مصحوباً بـ BOM) لتتعرف عليه جميع إصدارات Excel تلقائياً باللغة العربية. تأكد من فتح الملف المصدر وحفظه بترميز UTF-8 في حال استمرار المشكلة.
